Prototypes

Three design directions for the AI Providers brief.

Same content, three visual registers. Each is a single-page demonstration of the design direction. Pick a winner; we'll build out the full four-page version in that direction.

Direction 1

Consulting executive brief

McKinsey / BCG / federal-report register

Formal, structured, numbered sections (1.0 / 2.1). Executive-summary block up top. Conservative IBM Plex typography, Babson green for hierarchy, charcoal for body. 2×2 cost-vs-adoption matrix in pure CSS grid. A page that looks like a printed deliverable handed across a desk.

Stack: Vanilla HTML/CSS, IBM Plex Serif/Sans/Mono, no JS, no chart library, print-friendly.

Direction 2

Dashboard / data product

Stripe Atlas / Linear / modern internal-tool register

Information-dense, tool-feel. Sticky 240px left rail with scroll-spy nav. Five-card KPI strip with animated count-up. Per-user pricing bar chart + cumulative peer-deployment line chart. Tabbed filter on provider cards. Inter + JetBrains Mono accents on every number, date, and URL.

Stack: Vanilla HTML/CSS/JS with Chart.js 4.4 via CDN, Inter + JetBrains Mono via Google Fonts.
Open
Direction 3

Magazine editorial

FT / The Atlantic / Stripe Press register

Long-form magazine feature. Fraunces display serif at large sizes for headings, Source Serif 4 for body. Oxblood editorial accent on ivory paper. Drop cap, four numbered sections (I–IV), pull-quotes with hairline rules, oversized inline stats ($17M as typographic art). No charts; prose-led. Ends with a § endmark.

Stack: Vanilla HTML/CSS, Fraunces + Source Serif 4 + Inter via Google Fonts, no JS, no chart library.
